# 출퇴근 관리 시스템 — Render.com 배포 가이드

## 📁 파일 구조
```
출퇴근서버앱/
├── app.py              ← Flask 백엔드
├── requirements.txt    ← Python 패키지 목록
├── render.yaml         ← Render 배포 설정
└── static/
    └── index.html      ← 프론트엔드 (웹 화면)
```

---

## 🚀 배포 순서 (약 15분)

### 1단계 — GitHub 계정 준비
1. https://github.com 에서 무료 계정 만들기 (이미 있으면 생략)

### 2단계 — GitHub에 폴더 올리기
1. https://github.com/new 접속
2. Repository 이름 입력 (예: `attendance-app`) → **Create repository**
3. **"Upload files"** 클릭
4. 이 폴더 안에 있는 파일 4개 + `static/index.html` 모두 드래그&드롭
   - `app.py`
   - `requirements.txt`
   - `render.yaml`
   - `static/index.html` (static 폴더째로)
5. **Commit changes** 클릭

### 3단계 — Render.com 배포
1. https://render.com 접속 → **Get Started for Free** (GitHub 계정으로 로그인)
2. 대시보드에서 **+ New** → **Web Service** 클릭
3. GitHub repository 선택 (방금 만든 `attendance-app`)
4. 설정 확인:
   - **Name**: attendance-app (자유롭게)
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app`
   - **Instance Type**: Free ✅
5. **Create Web Service** 클릭
6. 약 2~3분 후 배포 완료 → URL 생성됨

### 4단계 — URL 공유
배포 완료 후 Render가 URL을 제공합니다:
```
https://attendance-app-xxxx.onrender.com
```
이 URL을 직원들에게 공유하면 됩니다! 🎉
